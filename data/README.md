# Motif Data — Canonical Source

This directory holds the **canonical motif database** for the Te Pā ecosystem and any sibling
or downstream Indigenous-culture site that wants to consume it.

## Files

| File | Purpose |
|---|---|
| `motif-bank-master.json` | The single source of truth: every motif, every culture, every translated meaning, colour palette, and R2 asset key. |
| `motif-course-map.json` | Links motifs to course modules / lessons (used by the calendar repo). |
| `campaign-categories.json` | Campaign / political-theme taxonomy used by `/api/motifs?theme=…`. |
| `social-calendar.json` | Per-motif social-post scheduling metadata. |

## Schema — `motif-bank-master.json`

```jsonc
{
  "version": "1.0",
  "total": 30,
  "cultures": [/* culture slugs */],
  "motifs": [
    {
      "id": "koru",                       // stable slug, [a-z0-9_]
      "culture": "maori",                 // canonical culture slug
      "language_primary": "te_reo_maori", // ISO-ish locale stem
      "region": "maori",                  // continental/cultural region
      "rhizome_region": "maori",          // used by Rhizome network grouping
      "name_en": "Koru",
      "name_indigenous": "Koru",
      "form_geometry": "unfurling spiral, …",
      "visual_description": "A teardrop opening into …",
      "colour_primary": "#2ecc71",
      "colour_secondary": "#1a9e55",
      "colour_traditional": "ochre, charcoal, red oxide, …",
      "meaning_en": "…",
      "meaning_pt": "…",       // Portuguese
      "meaning_sm": "…",       // Samoan
      "meaning_gn": "…",       // Guaraní
      // meaning_ar / meaning_mi may also appear
      "resistance_use": "Used in education campaigns, …",
      "political_themes": ["renewal", "education", "language", …],
      "svg_r2_key": "motifs/maori/koru.svg",
      "sources": ["ANZAAE kowhaiwhai guide", "Te Papa collections", …]
    }
  ]
}
```

### Cultures (11)

`maori`, `samoan`, `pacific`, `tongan`, `fijian`, `guarani`, `shipibo`, `guna`, `kayapo`,
`yanomami`, `amazonian`

### Languages with translated `meaning_*` fields

`en` (canonical), `mi` (te reo Māori), `sm` (Samoan), `pt` (Portuguese), `gn` (Guaraní),
`ar` (Arabic). Missing translations are documented in [`docs/i18n-audit.md`](../docs/i18n-audit.md).

## Consuming this data

There are three supported paths:

### 1. Public HTTP API (recommended for external sites)

Fully documented at [`docs/MOTIF-API.md`](../docs/MOTIF-API.md). Production base URL:

```
https://te-pa-analytics.sketchschool.workers.dev/api
```

No auth required for read endpoints. Returns JSON with `attribution`, `license`, and
`generated_at` envelope fields on every response.

### 2. Direct JSON download

```bash
curl -L https://te-pa.org/data/motif-bank-master.json -o motif-bank-master.json
```

Mirrored from this directory at every deploy.

### 3. Fork & self-host

See [`SELF-HOST-FORK.md`](../SELF-HOST-FORK.md) (PR 9). Use this if you are a community
that wants to maintain your own copy with your own motifs, colours, and language strings.

## Asset paths

Every motif has matching artwork at these R2 paths:

```
https://pub-bf8eea881c1e44d88eda5192c3b92291.r2.dev/motifs/{culture}/{id}.svg
https://pub-bf8eea881c1e44d88eda5192c3b92291.r2.dev/motifs/{culture}/{id}.png
```

Local SVG copies (for offline / sibling-site bundling) live in
[`../assets/motifs/`](../assets/motifs/) with a manifest at
[`../assets/motifs/manifest.json`](../assets/motifs/manifest.json).

## Contribution flow

1. Edit `motif-bank-master.json` in a feature branch.
2. Run `python3 scripts/audit-translations.py` — make sure no new keys regress non-Māori
   locales further than the existing audit baseline.
3. Validate JSON: `python3 -m json.tool data/motif-bank-master.json > /dev/null`.
4. If you add a new motif: ensure `svg_r2_key` is reachable **and** add a local SVG to
   `assets/motifs/{id}.svg` (the manifest is regenerated by the build).
5. Open a PR — the Worker will start serving the new motif the moment the change lands
   on `main` (Worker reads from a packaged copy at deploy time).
